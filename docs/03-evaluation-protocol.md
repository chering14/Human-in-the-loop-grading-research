# 评测协议

## 总体原则

评测必须先验证教师基准，再评价模型。若教师间一致性本身不足，模型与单一教师分数接近并不代表可靠。所有阈值、指标、缺失数据处理和排除规则应在测试前预注册，避免事后选择有利口径。

## 阶段A：离线验证

阶段A使用历史作业或合成回放环境，不向学生发送模型结果。数据应划分为校准集和测试集。校准集用于确定题型阈值、复核规则和严重错误定义；测试集只用于一次性报告。

主观题应由两名教师独立评分。若两者差异超过预注册阈值，第三名教师进行裁定。报告顺序为：教师间一致性、裁定分形成规则、模型相对裁定分的误差。建议报告二次加权 Kappa 或组内相关系数，以及模型 MAE、RMSE、偏差和容许误差命中率。

客观题报告模型原始正确率、自动通过覆盖率、自动通过错误率、错误捕获率、严重错误逃逸率、复核负担和复核后最终正确率。自动通过覆盖率越高并不必然越好，必须与自动通过错误率和严重错误逃逸率共同解释。

## 阶段B：真实教学验证

阶段B以教师-作业批次为基本分析单元，避免把同一班级同一次作业中的多道题当作独立样本。若采用阶梯楔形设计，应随机化上线顺序，并用混合效应模型控制学校、教师、班级和时间段差异。

主要结局建议设为教师净处理时长和订正后等值测验表现。次要结局包括订正完成率、学生申诉率、教师采纳率、复核负担、面批命中率和延迟保持测验。分析应采用意向性处理原则，明确缺失数据处理方法。

## 样本量与安全门槛

零错误样本不能直接证明系统足够安全。若独立样本中未观察到错误，约 149 个零错误样本才足以使单侧 95% 下界与 98% 准确率相容；若要把严重错误率上界约束到 0.1% 以下，则需要约 2995 个零严重事件样本。真实教育数据存在班级和教师聚类，实际所需样本量通常更大。

## 本仓库指标口径

`analysis/evaluate.py` 只用于演示以下计算：

- 客观题原始正确率及 Wilson 区间；
- 主观题教师间二次加权 Kappa；
- 模型相对裁定分的 MAE、RMSE、偏差和容许误差命中率；
- 自动通过覆盖率、自动通过错误率、错误捕获率、严重错误逃逸率；
- 复核负担、复核后最终正确率和描述性时间差异。

由于样本为合成数据，脚本输出中的 `deployment_authorized` 固定为 `false`。
