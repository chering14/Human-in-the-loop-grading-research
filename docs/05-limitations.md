# 局限性

## 概念设计限制

本仓库没有接入真实作业图像、学校系统、教师评分流程或学生订正数据。交互原型仅用于展示复核队列和证据呈现方式，不能证明实际可用性、教师接受度或学习效果。

## 合成数据限制

合成样本数量很小，且由研究者构造，不能代表真实学校中的题型分布、书写质量、评分差异、课堂环境或学生群体差异。任何由合成样本计算出的准确率、节时比例或风险指标都只能解释为代码演示结果。

## 模型限制

作业批改涉及手写识别、版面解析、学科推理、评分量表理解和反馈生成。任一环节出错都可能影响最终结果。尤其是主观题和步骤题，模型可能给出看似合理但证据不足的解释，因此必须保留教师复核。

## 推广限制

不同地区、学校、年级、教材版本和教师评分习惯会影响系统阈值。一个学校中校准出的分流规则不能直接推广到另一学校。跨校推广前必须重新校准，并报告群体公平性和错误分布。

## 需要进一步研究的问题

后续研究应重点回答四个问题：风险分流能否在真实工作流中降低教师净负担；证据型反馈能否改善订正质量；系统是否对不同学生群体产生差异化错误；教师修订如何在不侵犯学生权益的条件下改进校本规则。
